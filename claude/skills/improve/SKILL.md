# 改善提案スキル

## Description

`~/.claude/friction-log.md` に蓄積した観察を読み、パターンを分析して具体的な改善提案を出す。
改善内容はdotfiles（CLAUDE.md、スキル、hook）への反映まで行う。

TRIGGER when: `/improve`、"改善"、"フリクション確認"

## User Invocable

yes

## Instructions

### 分析の手順

1. **データを収集する**
   - `~/.claude/friction-log.md` を読む（存在しない場合はスキップ）
   - `~/.claude/usage-data/facets/` の最新JSONファイルを読む（最大30件）
     - 各ファイルの `friction_detail`、`friction_counts`、`underlying_goal`、`outcome` を参照する

2. **パターンを分析する**
   - 同じ種類の `[FRICTION]` が複数回出ていないか
   - `[IDEA]` に共通するテーマはあるか
   - 改善効果が高そうな（繰り返し発生している）問題はどれか

3. **改善提案を出す（優先順位付き）**
   各提案には以下を含める:
   - 問題パターン（何回発生しているか）
   - 提案内容（CLAUDE.mdルール追加 / 新スキル作成 / hook追加 / 既存スキル改善）
   - 具体的な変更内容のドラフト

4. **ユーザーに確認し、承認されたものをdotfilesに反映する**
   - 具体的な情報は抽象化し、汎用的なルールとして反映する
   - 反映後はコミットまで行う（`/cmp` を使う）

### 出力フォーマット例

```
## 改善提案

### 1. [繰り返し発生：3回] PR作成時のプレフィックス確認漏れ
- 問題: PR作成前にtitle prefixの確認をせずに進むことがある
- 提案: `/pr` スキルに「必ず確認」の強調を追加
- ドラフト: （具体的な変更内容）

### 2. ...
```
