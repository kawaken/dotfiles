# セッション振り返りスキル

## Description

セッション終了時にやり取りを振り返り、気づき（friction・良かった点・改善アイデア）を `~/.claude/friction-log.md` に記録する。

TRIGGER when: `/retro`、"振り返り"、"ふりかえり"

## User Invocable

yes

## Instructions

### 振り返りの手順

1. **セッションを振り返る**
   - ユーザーが修正・やり直しを指示した場面（friction）
   - うまくいった場面（良かった点）
   - 「こうすればよかった」と思った場面（改善アイデア）

2. **カテゴリ分類**
   - `[FRICTION]` — Claudeの挙動でユーザーが修正・やり直しを求めた場面
   - `[GOOD]` — うまく機能した場面
   - `[IDEA]` — 改善アイデア（CLAUDE.mdルール、新スキル、hook等）

3. **`~/.claude/friction-log.md` に追記**
   ```markdown
   ## YYYY-MM-DD [プロジェクト名]

   - [FRICTION] 内容（具体的に何が起きたか）
   - [GOOD] 内容
   - [IDEA] 内容
   ```
   - プロジェクト名は現在の作業ディレクトリから判断する
   - ファイルが存在しない場合は新規作成する
   - 既存の内容は保持し、先頭に追記する

4. **追記した内容をユーザーに見せて確認する**

### 注意
- 仕事の具体的な内容（コード、社名、機能名等）は含めず、パターンとして抽象化して記録する
- 簡潔に。1項目1-2文程度
