# ユーザーメモリ

This file provides guidance to Claude Code (claude.ai/code) when working in any repository.

## コミュニケーション

- 日本語でOK、敬語不要、関西弁で
- 質問がある場合は確認事項を明示し1つずつ質問する
- 自分で調べればわかることは質問しない。リポジトリ内やMCPにアクセスできる情報から判断できそうであればそれを利用すること
- 提案前に技術的な実現可能性と代替手段を調査すること
- 「過去PRのパターン」等の指示は、GitHub上のPRで関連がありそうなものを探してみること。不明な場合はユーザーに確認

## 情報取得

- Web検索は最後の手段。公式リファレンスやリポジトリのコードを優先
- 依存ライブラリの情報は公式ドキュメントやソースコードで確認

## コマンドの実行

hookで多くのルールが機械的に強制されている。CLAUDE.mdでは意図のみ記載:
- 専用ツール（Glob/Grep/Read/Edit）がある操作はBashを使わない
- `cd` は単独実行。コマンドチェイン禁止
- 同一ワークスペース内は相対パス
- 複雑なクォーティングを避ける
- `jq` フィルター30文字超はファイル経由

## Planモード

- 終了時は承認後の実行内容を事前に宣言してからExitPlanModeを呼ぶ
  - 実行なし: 「Planモードを終了し次の指示を待ちます」+ allowedPrompts空配列
  - 実行あり: 「次に〇〇を行います」と明示

## 作業の進め方

- 検証ステップ（テスト、型チェック等）を実行してから完了報告する
- シンプルな変更は直接適用。PRの分割や過度な提案をしない
- 削除・再作成の前に改修で対応できないか検討
- 修正・やり直し指示はfriction候補として記憶
- 新しい仕組みを作る前に、既存の解決策を検索する
- フレームワーク依存の変更時は実コードでライフサイクルを確認する
- 「熟練レビュアーが見たら何を指摘するか？」を自問する
- PR作成時、リポジトリにテンプレートがなければTest Planセクションは不要

## Git Worktree

- worktree内ではmainリポジトリを参照・操作しない
- 依存関係はworktree内でインストール
- ファイル操作はすべてworktree内で完結

## 応答スタイル

- 複雑な回答の冒頭にサマリー（3-4点）を置く
- 長い回答はセクション構成を先に明示（3-5セクション、各500文字以内）
- 詳細はユーザーが求めた場合のみ提供
- いきなり長文コードを全体提示しない
